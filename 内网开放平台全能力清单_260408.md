# 内网开放平台全能力清单（可落地版）

> 基于主流开放平台最佳实践、开源成熟实现、企业内网安全规范整合而成，专为公司内部业务线、开发团队提供能力共享、系统互联互通、避免重复建设的公共服务平台
> 调研范围：参考业界主流开放平台架构设计，适配公司内部系统打通、能力复用的实际场景

***

## 一、能力优先级划分

| 优先级    | 说明            |
| :----- | :------------ |
| **P0** | 基础核心能力，必须优先实现 |
| **P1** | 增强体验能力，中期落地   |
| **P2** | 生态扩展能力，长期建设   |

## 三、落地实施建议

### 1. 实施路径

| 阶段   | 核心目标     | 重点建设能力                                      |
| :--- | :------- | :------------------------------------------ |
| 第一阶段 | 最小可用版本上线 | Layer 0基础设施层 + Layer 1核心业务层 + Layer 4安全合规能力 |
| 第二阶段 | 成熟可用版本   | Layer 2开发者运营服务 + Layer 3企业客户管理能力            |
| 第三阶段 | 生态化版本    | Layer 5生态扩展能力（连接器+低代码+AI）                   |

### 2. 内网适配优化建议

- **优先开放内部公共核心能力**：优先开放组织架构、消息中心、审批、待办等公司级公共能力，以及各业务线可复用的通用能力，避免重复建设
- **API设计以内部开发者为中心**：不要照搬内部接口逻辑，站在使用方视角设计易用的接口，降低内部系统集成成本
- **安全左移**：所有能力上线前先考虑安全管控规则，符合内网数据安全规范，敏感数据访问严格控制
- **小步快跑迭代**：先上线最小可用版本，根据内部团队反馈快速优化，快速响应内部业务需求
- **对接内部统一体系**：所有能力对接内部统一身份认证、统一权限、统一审计体系，符合公司整体IT规范
