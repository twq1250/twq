# IT平台调研报告：现有平台与飞书、钉钉开放平台能力对比分析及未来发展建议

## 一、调研背景与目的

本次调研旨在对比分析现有平台与飞书、钉钉开放平台的能力差异，识别差距并提出未来发展建议，为平台能力增强提供参考方向。

***

## 二、现有平台能力概述

现有平台已具备以下核心能力：

| 能力模块      | 具体功能                         |
| --------- | ---------------------------- |
| **应用管理**  | 应用创建、应用基本信息维护、成员管理           |
| **卡片能力**  | 群置顶、群通知、链接增强、点对点通知、助手广场卡片    |
| **API管理** | 24个API接口，负责API的开放与管理         |
| **事件管理**  | MQS平台订阅、将事件发送至业务系统、回调调用指定URL |
| **运维支持**  | 操作日志、版本发布                    |
| **文档支持**  | API/能力详细文档（文档展示）             |

### 2.1 当前已开放API（24个）

| API分类     | 具体能力                            |
| --------- | ------------------------------- |
| **会议能力**  | 会议创建、会议修改、会议查询、会议取消             |
| **卡片能力**  | 卡片创建、卡片更新、卡片推送                  |
| **公众号能力** | 公众号查询                           |
| **群组能力**  | 群组查询成员、修改群组头像、群角色查询、群成员校验、群角色绑定 |

### 2.2 当前事件订阅方式

| 方式        | 说明           |
| --------- | ------------ |
| MQS平台订阅   | 通过消息队列平台订阅事件 |
| 推送至业务系统   | 将事件发送至开发者服务器 |
| 回调调用指定URL | 支持回调URL配置    |

## 三、飞书开放平台能力分析

### 3.1 核心能力概览

飞书开放平台提供**2500+个标准化服务端API接口**，涵盖以下核心模块：

| 能力模块  | API数量 | 事件数量 |
| ----- | ----- | ---- |
| 身份验证  | 11    | 1    |
| 通讯录   | 78    | 17   |
| 消息与群组 | 75    | 14   |
| 云空间   | 40    | 10   |
| 文档    | 12    | -    |
| 电子表格  | 56    | -    |
| 多维表格  | 52    | 2    |
| 知识库   | 16    | -    |
| 日历    | 44    | 4    |
| 视频会议  | 58    | 18   |
| 会议室   | 17    | 5    |
| 考勤打卡  | 39    | -    |
| 审批    | 55    | 7    |
| 服务台   | 50    | 4    |
| 任务    | 75    | 3    |
| 工作台   | 3     | -    |
| 邮箱    | 67    | 1    |

### 3.2 应用形态

飞书支持三种应用形态：

1. **机器人（Bot）**：基于会话与用户交互，支持自动化管理群组、响应消息
2. **网页应用（H5）**：运行在飞书客户端内的H5应用，支持SSO免登集成
3. **小组件（Block）**：轻量级功能块，嵌入云文档或多维表格中

### 3.3 事件订阅机制

飞书提供两种事件订阅方式：

- **长连接模式（推荐）**：通过WebSocket全双工通道接收事件，SDK内封装鉴权逻辑，开发周期从1周缩短至5分钟
- **Webhook模式**：传统HTTP POST方式，需提供公网地址，支持AES-256-CBC加密

### 3.4 特色能力

- **云文档小组件**：在飞书云文档中嵌入自定义功能模块
- **多维表格插件**：在多维表格中嵌入自定义插件
- **网页组件**：在企业自有系统中嵌入飞书功能（云文档、成员名片等）
- **开放平台智能助手**：AI驱动的开发辅助工具，支持概念解释、方案设计、报错诊断

### 4.2 事件推送机制

钉钉提供三种事件推送方式：

| 方式               | 特点                        | 适用场景          |
| ---------------- | ------------------------- | ------------- |
| **Stream模式（推荐）** | WebSocket连接，无需公网服务器、IP、域名 | 降低接入门槛，本地开发调试 |
| **SyncHTTP推送**   | HTTP POST方式，加密传输，业务数据最终状态 | 本地部署场景        |
| **RDS推送**        | 阿里云RDS数据推送，安全等级高、效率高      | 第三方ISV应用      |
| HTTP推送（不推荐）      | 早期方式，公网通道，安全性较低           | 仅支持企业内部应用     |

- 小程序应用
- 网页应用
- **酷应用**
- 机器人应用
- Webhook机器人
- 移动客户端接入

### 4.4 特色能力

- **连接平台**：提供连接器中心和自动化流程能力
- **AI能力**：AI藏头诗、智能需求收集等
- **宜搭**：低代码构建平台

## 五、能力对比分析

### 5.1 能力差距矩阵

| 能力维度        | 现有平台            | 飞书                  | 钉钉                      | 差距等级   |
| ----------- | --------------- | ------------------- | ----------------------- | ------ |
| **API规模**   | 24个             | 2500+               | 2000+                   | 较大     |
| **网页应用**    | 无               | 支持H5应用嵌入            | 支持网页应用                  | **较大** |
| **小组件/酷应用** | 无               | 小组件                 | 酷应用                     | **较大** |
| **会议能力**    | 4个API           | 58 API + 18事件       | 40+ API                 | 中等     |
| **卡片能力**    | 13个API          | 丰富卡片交互              | 互动卡片                    | 中等     |
| **群组能力**    | 6个API           | 75 API + 14事件       | 80+ API                 | 较大     |
| **公众号能力**   | 1个API           | -                   | -                       | 较小     |
| **事件订阅**    | MQS订阅 + Webhook | 长连接 + Webhook，SDK完善 | Stream + SyncHTTP + RDS | 较大     |
| **开发者工具**   | 文档展示            | API调试台、智能助手、SDK     | 开发助手                    | 较大     |

### 5.2 关键差距分析

#### 5.2.1 应用形态差距（核心差距）

- **现有平台**：仅支持API调用，不支持任何应用形态
- **飞书**：支持网页应用（H5）、机器人、小组件
- **钉钉**：支持网页应用、酷应用、机器人、小程序

#### 5.2.2 开发者工具差距

- **现有平台**：仅做文档展示
- **飞书**：API调试台、智能助手（AI问答）、多语言SDK、开发者社群
- **钉钉**：开发助手、连接平台自动化能力

#### 5.2.3 事件订阅机制差距

- **现有平台**：MQS订阅 + 推送至业务系统 + 回调URL
- **飞书**：提供长连接（WebSocket）模式，SDK支持完善，接入门槛低
- **钉钉**：Stream模式（WebSocket）、RDS推送等企业级方案

## 六、应用形态技术实现详解

### 6.1 网页应用（H5应用）

#### 6.1.1 飞书网页应用技术实现

**技术架构**：

- 前端：标准H5页面（HTML/CSS/JavaScript），可使用Vue、React等框架
- 后端：任意语言/框架（Node.js、Java、Python等），需提供公网可访问的HTTPS地址
- 通信：飞书JSAPI SDK + 服务端Open API

**实现流程**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         飞书网页应用免登流程                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 用户点击应用                                                             │
│       ↓                                                                    │
│  2. 飞书客户端加载H5页面                                                      │
│       ↓                                                                    │
│  3. 前端调用 JSAPI 获取授权码                                                │
│     window.tt.getUserInfo()                                                │
│       ↓                                                                    │
│  4. 前端将授权码发送给企业后端                                                │
│       ↓                                                                    │
│  5. 企业后端调用飞书 Open API                                                │
│     POST https://open.feishu.cn/open-apis/authen/v1/access_token           │
│     参数: app_id, app_secret, code                                          │
│       ↓                                                                    │
│  6. 飞书返回 user_access_token                                               │
│       ↓                                                                    │
│  7. 企业后端获取用户信息                                                     │
│     GET https://open.feishu.cn/open-apis/authen/v1/user_info              │
│       ↓                                                                    │
│  8. 企业后端创建会话，返回业务登录态                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键技术点**：

| 步骤      | 技术要点                                                                     |
| ------- | ------------------------------------------------------------------------ |
| 配置可信域名  | 在开发者后台配置H5可信域名，仅可信域名下的页面可调用JSAPI                                         |
| JSAPI注入 | 引入飞书JSAPI SDK：`https://sf3-cn.feishucdn.com/js-api-sdk/v1.0.1/feishu.js` |
| 免登实现    | 使用OAuth2.0授权码模式，前端获取code → 后端换取token → 获取用户信息                            |
| 权限申请    | 在开发者后台申请所需API权限（如contact:user.base:readonly）                             |

**开发步骤**：

1. 在飞书开发者后台创建企业自建应用
2. 开启网页应用能力，配置桌面端/移动端主页地址
3. 配置安全设置：重定向URL、H5可信域名
4. 申请所需API权限
5. 开发H5页面，接入JSAPI SDK
6. 实现免登流程
7. 发布应用

#### 6.1.2 钉钉网页应用技术实现

**技术架构**：

- 前端：标准H5页面（HTML/CSS/JavaScript）
- 后端：任意语言/框架
- 通信：钉钉JSAPI + 服务端API

**实现流程**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         钉钉网页应用免登流程                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 用户点击微应用                                                            │
│       ↓                                                                    │
│  2. 钉钉客户端加载H5页面                                                      │
│       ↓                                                                    │
│  3. 前端调用 JSAPI 获取授权码                                                 │
│     dingtalk.runtime.requestAuthCode()                                     │
│       ↓                                                                    │
│  4. 前端将授权码发送给企业后端                                                │
│       ↓                                                                    │
│  5. 企业后端调用钉钉 API                                                     │
│     POST https://api.dingtalk.com/v1.0/oauth2/getUserInfo                  │
│     参数: clientId, clientSecret, code                                      │
│       ↓                                                                    │
│  6. 钉钉返回用户userId                                                       │
│       ↓                                                                    │
│  7. 企业后端创建会话，返回业务登录态                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键技术点**：

| 步骤      | 技术要点                                                                        |
| ------- | --------------------------------------------------------------------------- |
| 配置安全域名  | 在开发者后台配置JSAPI安全域名                                                           |
| JSAPI注入 | 引入钉钉JSAPI：`https://g.alicdn.com/dingding/dingtalk-sdk/1.0.0.12/dingtalk.js` |
| 免登实现    | 使用`requestAuthCode`获取授权码，后端换取用户信息                                           |
| 签名校验    | 计算JSAPI签名（timestamp, nonce, signature）                                      |

**开发步骤**：

1. 在钉钉开发者后台创建企业内部应用
2. 添加网页应用能力，配置应用首页地址
3. 配置安全域名和重定向URL
4. 申请所需接口权限
5. 开发H5页面，接入JSAPI
6. 实现免登流程
7. 发布应用

#### 6.1.3 飞书与钉钉网页应用差异对比

| 对比维度        | 飞书                        | 钉钉                                   | 差异说明      |
| ----------- | ------------------------- | ------------------------------------ | --------- |
| **SDK名称**   | JSAPI SDK                 | 钉钉JSAPI                              | 命名不同      |
| **免登获取方式**  | `window.tt.getUserInfo()` | `dingtalk.runtime.requestAuthCode()` | API不同     |
| **Token获取** | OAuth2.0授权码模式             | 免登授权码模式                              | 流程略有差异    |
| **可信域名配置**  | 支持                        | 支持安全域名                               | 名称不同，功能类似 |
| **签名算法**    | 不需要JSAPI签名                | 需要JSAPI签名校验                          | 钉钉安全要求更高  |
| **开发工具**    | API调试台                    | 开发助手                                 | 工具不同      |

#### 6.1.4 调研平台实现建议

调研平台实现网页应用支持需要考虑以下方面：

**平台侧需要提供的支持**：

| 功能         | 说明                | 实现难度          |
| ---------- | ----------------- | ------------- |
| **应用配置管理** | 提供应用主页地址、可信域名配置界面 | 中             |
| **应用发布管理** | 应用状态管理（开发/测试/上线）  | 低             |
| **客户端嵌入**  | 需要聊天软件客户端支持H5页面渲染 | **高**（需客户端配合） |

**技术依赖**：

```
调研平台实现网页应用 ===================
        ↓
   需要聊天软件客户端支持
        ↓
   1. H5页面渲染能力（WebView）
        ↓
   2. JSAPI注入（获取用户信息、免登）
        ↓
   3. 应用入口展示（工作台/菜单）
```

**实现建议**：

- 调研平台本身不负责实现客户端能力，仅负责应用配置和展示
- 需要与聊天软件客户端团队协作，获取JSAPI支持
- 初期可先实现简单的H5页面嵌入，后续逐步完善免登等能力

### 6.2 小组件（飞书）/ 酷应用（钉钉）

#### 6.2.1 飞书小组件技术实现

**定位**：轻量级功能块，嵌入在云文档或多维表格中，实现文档与业务系统的数据打通

**技术架构**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         飞书小组件架构                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────────┐      ┌──────────────────┐      ┌──────────────────┐    │
│  │  飞书客户端   │      │   云文档/多维表格  │      │   业务系统后端   │    │
│  │  (渲染容器)  │ ←──→ │   (小组件容器)    │ ←──→ │   (数据源)       │    │
│  └──────────────┘      └──────────────────┘      └──────────────────┘    │
│         ↑                      ↑                        ↑              │
│         │                      │                        │              │
│    小组件前端代码          Blockit SDK              企业自有API          │
│    (Vue/React)           (数据交互)                (数据源)            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**开发流程**：

1. **创建应用**：在飞书开发者后台创建应用，开启小组件能力
2. **开发小组件**：使用飞书开发者工具开发小组件前端页面
3. **配置数据源**：在小组件中配置外部数据源URL
4. **发布小组件**：提交审核，审核通过后发布
5. **在文档中使用**：在云文档中插入小组件

**关键技术实现**：

| 功能       | 实现方式                                        |
| -------- | ------------------------------------------- |
| **数据绑定** | Button元素绑定外部数据源URL，页面加载时发起GET请求获取数据         |
| **交互触发** | 监听`onInlineComponentTriggerChange`事件，处理用户交互 |
| **数据更新** | 小组件通过API与业务系统通信，实时获取/提交数据                   |
| **组件渲染** | 使用Blockit SDK提供的组件库构建UI                     |

**示例代码结构**：

```javascript
// 小组件前端 - 监听数据加载
import { BlockitClient } from 'doc-mini-app-client';

const docMiniApp = new BlockitClient().initAPI();
docMiniApp.Events.onInlineComponentTriggerChange((event) => {
  // 处理用户交互，传递参数
  const params = event.value;
  // 调用业务系统API获取数据
});
```

#### 6.2.2 钉钉酷应用技术实现

**定位**：轻量级应用形态，无需下载、即点即用，强调与钉钉工作场景的深度集成

**技术架构**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         钉钉酷应用架构                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────────┐      ┌──────────────────┐      ┌──────────────────┐    │
│  │  钉钉客户端   │      │    酷应用容器     │      │   业务系统后端   │    │
│  │  (渲染容器)  │ ←──→ │  (小程序/H5)     │ ←──→ │   (数据源)       │    │
│  └──────────────┘      └──────────────────┘      └──────────────────┘    │
│         ↑                      ↑                        ↑              │
│    小程序框架                FaaS脚本                  企业自有API         │
│    (SSW/ARML)              (可选)                    (数据源)            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**开发流程**：

1. **创建应用**：在钉钉开发者后台创建应用，选择酷应用类型
2. **开发应用**：使用钉钉小程序开发工具或H5开发
3. **配置场景**：配置群聊酷应用、链接增强等场景
4. **发布应用**：提交审核，审核通过后发布到酷应用市场
5. **启用应用**：在群聊/工作台中启用酷应用

**酷应用类型**：

| 类型        | 场景     | 说明           |
| --------- | ------ | ------------ |
| **群聊酷应用** | 群会话侧边栏 | 在群聊中侧边栏展示应用  |
| **链接增强**  | 消息链接   | 点击链接在面板中展示应用 |
| **成员资料页** | 个人资料   | 在成员资料页展示应用   |

| **开发工具** | 飞书开发者工具         | 钉钉小程序开发工具/宜搭    | 工具不同              |
| **适用场景** | 文档协作、数据展示       | 业务审批、流程处理       | 场景侧重不同            |

#### 6.2.4 调研平台实现建议

调研平台实现酷应用/小组件 ===================

#### 7.1.2 API权限管理（高优先级）

调研平台，需支持API的分级权限管理：

**飞书/钉钉权限体系参考**：

| 维度       | 飞书                                                    | 钉钉                                            |
| -------- | ----------------------------------------------------- | --------------------------------------------- |
| **身份类型** | 应用身份（tenant\_access\_token）、用户身份（user\_access\_token） | 应用身份（access\_token）、用户身份（user\_access\_token） |
| **权限等级** | 普通权限（免审）、高级权限（需审核）                                    | 普通权限、高级权限                                     |
| **数据权限** | 可配置通讯录、组织架构等数据范围                                      | 可配置API调用范围                                    |

**调研平台实现方案**：

| 功能         | 说明              | 实现方式                    |
| ---------- | --------------- | ----------------------- |
| **身份类型区分** | 区分应用身份调用和用户身份调用 | API调用时指定身份类型            |
| **权限粒度**   | 按API维度配置权限      | 每个API可配置是否需要应用身份/用户身份权限 |
| **权限申请**   | 开发者申请API使用权限    | 平台提供权限申请表单，管理员审批        |
| **数据范围**   | 可配置数据访问范围       | 需根据业务需求设计（如按部门、按人员）     |

**实现优先级**：

1. **第一阶段**：先实现身份类型区分（应用身份 vs 用户身份）
2. **第二阶段**：实现权限申请和审批流程
3. **第三阶段**：实现数据范围控制（可选，根据业务需求）

### 7.2 中期建议

#### 7.2.1 JSSDK开放支持

支持前端页面调用平台提供的JSSDK能力（区别于客户端提供的原生JSAPI，调研平台可提供封装的企业级JSSDK）：

**飞书/钉钉 JSSDK体系参考**：

| 维度         | 飞书                                                                             | 钉钉                                       |
| ---------- | ------------------------------------------------------------------------------ | ---------------------------------------- |
| **JSSDK包** | H5 JSAPI (`window.tt`对象) + aPaaS client SDK (`@byted-apaas/client-sdk/feishu`) | `dingtalk-jsapi` npm包 + `dd`对象           |
| **调用方式**   | 注入到WebView的`window.tt`对象                                                       | 引入npm包后使用`dd`对象                          |
| **核心能力**   | 获取用户信息、免登、分享、支付、扫码等                                                            | 获取免登码、分享、选择人员、扫码、支付等                     |
| **签名要求**   | H5应用需要签名验证                                                                     | 需要JSAPI签名校验（timestamp, nonce, signature） |
| **应用场景**   | 网页应用、小程序                                                                       | 小程序、H5微应用                                |

**飞书JSSDK详情**：

```
引入方式：
- 直接使用：window.tt.getUserInfo() (需客户端注入)
- aPaaS方式：import { scanCode } from '@byted-apaas/client-sdk/feishu'
```

**钉钉JSSDK详情**：

```
引入方式：
- 小程序：import 'dingtalk-jsapi/entry/mobile'
- H5：<script src="https://g.alicdn.com/dingding/dingtalk-sdk/1.0.0.12/dingtalk.js"></script>
调用：dd.getAuthCode(), dd.chooseImage() 等
```

**调研平台JSSDK实现方案**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         调研平台JSSDK架构                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────────┐      ┌──────────────────┐      ┌──────────────────┐    │
│  │  H5页面      │      │   调研平台       │      │   业务后端       │    │
│  │(开发者开发)   │ ←──→ │  (JSSDK封装)     │ ←──→ │   (业务数据)     │    │
│  └──────────────┘      └──────────────────┘      └──────────────────┘    │
│         ↑                      ↑                        ↑              │
│    调研平台JSSDK            服务端API                企业自有API          │
│    (department-sdk.js)     (能力聚合)                (数据源)            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**调研平台JSSDK定位**：

| 层级            | 说明                                 |
| ------------- | ---------------------------------- |
| **原生JSAPI**   | 聊天软件客户端提供的能力（如扫码、相册、位置等）           |
| **调研平台JSSDK** | 封装调研平台API能力，供开发者在前端调用（可独立于原生JSAPI） |

**JSSDK能力规划**：

| 优先级   | JSSDK能力   | 说明                    |
| ----- | --------- | --------------------- |
| **高** | **免登授权**  | 获取用户授权码，用于后端换取用户身份    |
| **高** | **用户信息**  | 获取当前用户基本信息（姓名、头像、部门等） |
| **高** | **API代理** | 封装调研平台的API调用能力，前端直接调用 |
| **中** | **通讯录选择** | 弹窗选择部门内人员/部门          |
| **中** | **消息推送**  | 推送消息到当前会话             |
| **中** | **应用跳转**  | 跳转到其他应用               |

**实现建议**：

1. **优先实现封装SDK**：封装调研平台的服务端API，提供统一的JavaScript SDK
2. **支持混合模式**：根据需要可调用原生JSAPI（如扫码、相册等）
3. **发布npm包**：可通过`npm install @department/sdk`引入
4. **文档完善**：提供完整的API文档和示例代码

**与API权限管理的关系**：

- JSSDK前端调用需要权限控制，底层依赖API权限管理机制
- JSSDK中的每个能力应对应API的权限配置
- **建议优先级**：API权限管理 > JSSDK开放

#### 7.2.6 MCP开发平台

**MCP协议价值**：

```
传统方式：AI工具 → 单独开发适配器 → 企业API
MCP方式：AI工具 → 标准MCP协议 → 企业API

优势：统一标准、即插即用、降低对接成本
```

**调研平台MCP实现方案**：

│         ↑                      ↑                        ↑              │
│    MCP协议                  API包装                  企业自有API          │
