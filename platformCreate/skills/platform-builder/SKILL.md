# Platform Builder - 平台搭建器

## 技能概述

这个技能用于根据用户提供的平台规格文档、前端设计 skill 文档和流程 agents.md，自动分析并搭建完整的平台网站。

## 使用场景

当用户需要：
- 根据已有的平台规格文档搭建网站
- 使用指定的前端设计 skill 进行开发
- 遵循特定的流程 agents.md 执行搭建流程

## 执行流程

### Phase 1: 文档收集与确认

**必须要求用户提供以下文档：**

1. **平台规格文档** (`platform-spec.md` 或类似)
   - 平台目录结构
   - 页面布局规范
   - 交互设计规范
   - 样式规范

2. **前端设计 skill 文档** (`frontend-design-skill.md` 或类似)
   - 使用的设计系统
   - UI 组件库
   - 样式方案

3. **流程 agents.md** (`agents.md` 或类似)
   - 搭建流程定义
   - 执行步骤
   - 质量检查点

### Phase 2: 文档分析与理解

1. **读取并分析平台规格文档**
   - 解析平台目录结构
   - 理解页面布局规范
   - 提取交互和样式要求

2. **读取前端设计 skill 文档**
   - 确认使用的设计系统
   - 了解 UI 组件库
   - 掌握样式方案

3. **读取流程 agents.md**
   - 理解搭建流程
   - 明确执行步骤
   - 确认质量检查点

### Phase 3: 细节确认

**与用户逐个确认以下细节：**

1. **目录结构确认**
   - 确认项目根目录结构
   - 确认各功能模块位置
   - 确认资源文件存放位置

2. **页面布局确认**
   - 确认每个页面的布局结构
   - 确认导航栏、侧边栏、页脚等公共组件
   - 确认响应式设计要求

3. **交互细节确认**
   - 确认用户交互行为
   - 确认动画效果
   - 确认加载状态和错误处理

4. **样式细节确认**
   - 确认颜色方案
   - 确认字体规范
   - 确认间距和尺寸规范

### Phase 4: 按流程搭建

**严格按照流程 agents.md 执行：**

1. **初始化项目**
   - 创建项目目录结构
   - 初始化配置文件
   - 安装依赖

2. **搭建基础架构**
   - 配置路由
   - 创建布局组件
   - 设置状态管理

3. **实现页面组件**
   - 按照设计文档创建页面
   - 实现交互逻辑
   - 应用样式

4. **集成与测试**
   - 集成各个组件
   - 运行测试
   - 修复问题

### Phase 5: 验收与交付

1. **自检**
   - 对照规格文档检查功能
   - 对照设计文档检查样式
   - 运行所有测试

2. **用户验收**
   - 提供验收清单
   - 引导用户测试
   - 修复反馈问题

3. **交付**
   - 提交代码
   - 提供部署说明
   - 提供使用文档

## 行为规则

1. **文档驱动**
   - 必须基于用户提供的文档进行开发
   - 不要自作主张添加文档中没有的功能
   - 遇到文档不明确的地方必须询问用户

2. **流程遵循**
   - 严格按照流程 agents.md 执行
   - 每个步骤完成后必须确认
   - 遇到问题及时沟通

3. **细节确认**
   - 每个设计细节都要与用户确认
   - 不要假设用户的偏好
   - 提供专业建议但尊重用户选择

4. **质量保证**
   - 每个组件都要有测试
   - 代码要符合设计规范
   - 性能要满足要求

## 输出物

- 完整的平台网站代码
- 测试文件
- 部署配置文件
- 验收报告

## 注意事项

- 在用户确认所有细节之前，不要开始编写实现代码
- 严格按照流程 agents.md 执行，不要跳过任何步骤
- 使用用户指定的前端设计 skill，不要擅自更换
- 每个阶段完成后都要与用户确认
